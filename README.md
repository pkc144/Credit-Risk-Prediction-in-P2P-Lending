# Credit Risk Prediction in P2P Lending

## Overview
Predict **borrower default** in Peer-to-Peer (P2P) lending using the Bondora
loan dataset (2012–2016). The project compares six classical ML models with a
hybrid SHAP + model-importance feature-selection step, and produces a full set
of evaluation artifacts (metrics table, confusion matrices, ROC curves, top-15
feature plots) under `reports/figures/`.

---

## Features
- End-to-end preprocessing of the raw Bondora CSV
- Class balancing via oversampling on the training fold
- Hybrid feature selection (SHAP ∪ model `feature_importances_`)
- Six baselines: Logistic Regression, Decision Tree, Random Forest, XGBoost,
  LightGBM, CatBoost
- Reproducible artifacts (plots, CSV metrics, persisted `.joblib` models)

## Getting Started

### 1. Clone & set up
```bash
git clone https://github.com/pkc144/Credit-Risk-Prediction-in-P2P-Lending.git
cd Credit-Risk-Prediction-in-P2P-Lending
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

> **macOS note**: XGBoost and LightGBM require OpenMP. If `import xgboost`
> fails with `Library not loaded: libomp.dylib`, install it via Homebrew:
> `brew install libomp`.

### 2. Preprocess the raw data
```bash
python -m src.data.preprocessing --input data/raw/trainset.csv --output data/processed/cleaned_p2p.csv
```

### 3. Train and evaluate all models
```bash
python -m src.models.train --data data/processed/cleaned_p2p.csv
```
Or train a subset:
```bash
python -m src.models.train --only XGBoost LightGBM CatBoost
```

### 4. (Optional) Rank features with SHAP for a saved model
```bash
python -m src.features.shap_features --model models/xgboost.joblib
```

### 5. (Optional) Per-model standalone scripts
Each model has three notebook-style scripts under `src/models/variants/`:
`<model>_basic.py` (importance only), `<model>_shap.py` (SHAP only),
`<model>_shap_union.py` (hybrid). They were authored on Kaggle and contain
hard-coded `/kaggle/input/...` paths — edit those before running locally.

All plots and the metrics table land under `reports/figures/`.

## Pipeline

1. **Preprocessing** (`src/data/preprocessing.py`)
   - Drop ID/leaky columns (`UserName`, `LoanNumber`, `PrincipalBalance`, `Status`)
   - Drop fully-empty columns (e.g. `EmploymentPosition`)
   - Mode-fill object columns, mean-fill numeric columns
   - `LabelEncoder` on categoricals
   - Winsorise numeric columns at the 0.1% / 99.9% quantiles
   - Drop one of each pair of features with `|corr| > 0.95`

2. **Training** (`src/models/train.py`)
   - Stratified 80/20 split, oversample minority class on train fold
   - Fit each model, take top-15 features by importance, refit on the slice
   - Persist model + selected feature list under `models/`

3. **Evaluation** (`src/evaluation/evaluate.py`)
   - Accuracy, Precision, Recall, F1, AUC
   - Confusion matrix, ROC curve, top-15 importance plot per model
   - Combined metrics table (`reports/figures/evaluation_metrics/`)

## Results

Latest run on the cleaned 24,000-row dataset (80/20 stratified split, top-15
features by built-in importance refit):

| Model        | Accuracy | Precision | Recall | F1     | AUC    |
|--------------|----------|-----------|--------|--------|--------|
| LogReg       | 0.6987   | 0.6968    | 0.7037 | 0.7002 | 0.7646 |
| DecisionTree | 0.7583   | 0.7329    | 0.8129 | 0.7708 | 0.7583 |
| **RandomForest** | **0.8117** | **0.7829** | **0.8625** | **0.8208** | **0.9131** |
| XGBoost      | 0.7598   | 0.7291    | 0.8267 | 0.7748 | 0.8373 |
| LightGBM     | 0.7608   | 0.7234    | 0.8446 | 0.7793 | 0.8388 |
| CatBoost     | 0.7573   | 0.7291    | 0.8187 | 0.7713 | 0.8363 |

Best model: **Random Forest** (AUC 0.913, F1 0.821). Plots for every model
are checked in under `reports/figures/`; re-running
`python -m src.models.train` overwrites them and rewrites
`evaluation_metrics.csv`.
